## Что нового

### Поддержка стандартных TLS-серверов

Раньше приложение поддерживало только VLESS-серверы с технологией REALITY. Теперь можно добавлять серверы с обычным TLS (`security=tls`) — это открывает совместимость с большим количеством VPN-провайдеров.

**Что было:** при добавлении ссылки с `security=tls` xray выдавал ошибку *"Failed to build REALITY config — empty password"*, потому что парсер всегда генерировал конфиг для REALITY, даже когда в ссылке указан обычный TLS.

**Что стало:** парсер читает параметр `security=` из ссылки и корректно генерирует:
- `tlsSettings` для обычного TLS (с поддержкой `alpn`, `fingerprint`, `allowInsecure`)
- `realitySettings` для REALITY (как и раньше)

### Поддерживаемые типы ссылок

| Тип | Пример | Статус |
|---|---|---|
| VLESS + TLS + TCP | Стандартные провайдеры | **Новое в v1.3.0** |
| VLESS + REALITY + Vision | gofizz и аналоги | Работает |
| VLESS + REALITY + XHTTP | e0f и аналоги | Работает |
| VLESS + TLS + XHTTP | Если появятся | **Новое в v1.3.0** |

Все три типа протестированы на реальном роутере с xray 26.3 — конфигурация проходит валидацию, соединения устанавливаются, данные передаются.

### Прочее
- `buildVisionOutbound` переименован в `buildTcpOutbound` — теперь обрабатывает все TCP-транспорты, а не только Vision
- Параметр `flow` больше не форсируется в `xtls-rprx-vision` для TLS-ссылок без flow
